# Seven Stud Hi-Lo Trainer（学習用・オフライン）

**Seven Card Stud Hi-Lo（8 or Better）** の学習用 Web アプリです。  
**実際のお金・換金・課金はありません。** すべて **換金不可の仮想チップ** です。外部 API やサーバー通信は使いません。

**React + Vite への移行作業**は `feature/react-ui-prototype` ブランチの `react-ui/` で進めています。引き継ぎ・現状整理は **`docs/REACT_MIGRATION_NOTES.md`** を参照してください。

## 画面構成（SPA・単一 `index.html`）

ブラウザを再読み込みすると **常にトップ画面** から始まります。JavaScript で `data-view` 相当の画面 ID（`top` / `preGame` / `game` / `rules` / `stats` / `history`）を切り替えています。

## デザイン方針

懐かしい **カジュアルなオンラインポーカー** の雰囲気（濃いグリーンのフェルト、木目／ダークブラウン系背景、ゴールド系アクセント、丸みのあるボタン）を **オリジナルの CSS** で表現しています。  
**特定サービスのロゴ・画像・UI 素材・名称のコピーは行っていません。**

## 進行・UI（2026 年アップデート）

### サイドポットとオールイン（要点）

- 各席は **チップ・そのラウンドの投入・ハンド通算の投入（totalCommitted）・フォールド・オールイン** を区別して管理します。
- **オールイン**した席は **フォールド扱いにしません**。以降のベットラウンドでは **アクション対象外**ですが、**カードは最後まで受け取り、ショーダウンに残ります**。
- **ショーダウン時**に、全席の `totalCommitted` から **メインポット／サイドポット** を構成し、**各ポットごとに eligible な席だけ**でハイ・ロー・スクープを判定して分配します。
- **フォールドした席**は **勝者になれません**が、**すでにポットに出したチップはポット総額に含まれたまま**です（没収分として上乗せ側のポットに回ります）。
- **オールインで短いスタック**の席は、**自分が支払った額までしか取れないポット**（メイン等）の eligible に含まれ、**それ以上の差額はサイドポット**で扱います。
- **再レイズの厳密なキャップやカジノ細則**は簡易です。重要なのは **ポットの整合**と **ショーダウンが破綻しないこと** です。

### ログの役割（下部ログ / ログタブ / 結果タブ）

- **下部ログ（ドック上段左）**: **ハンド終了後の各席チップ増減だけ**（ハンド開始時点のスタックとの差分。`第Nハンド結果` のあと `席：+N` / `席：-N`）。**プレイ中のアクション・フェーズ名・Main/Side/eligible 等は出しません**（誤って混ざらないよう `addResultDeltaLog` のみが追記）。ハンドごとに **点線区切り**。
- **ログタブ**: **詳細ログ**（アンティ・ブラインド・メインポット／サイドポット作成行、層別勝者、ガイド `[ガイド]` など全文）。`addDetailLog`（従来の `logLine` と同じ経路）。
- **結果タブ**: **ハイ／ロー勝者・ポット分配・サイドポット内訳**の確認用（手動で開く）。サイドポットの内訳は **ここで確認**。

### ゲーム画面（タブ・日本語 UI・チップ補充）

- **プレイ／ログ／結果／デバッグ**: 初期表示は **プレイ**。**ログ** タブにオールイン・ポット作成・ポット別勝者・アンティ／ブラインドなど**全文の詳細**を記録します。**ショーダウン完了やフォールドのみの決着後も、表示中がプレイならプレイタブのまま**（結果タブへ自動遷移しません）。
- **結果タブ**: **ポット内訳**（Main / Side ごとの金額・対象席・ハイ／ロー勝者・分配・スクープ）と合算サマリー。**手動でタブを開いたときだけ**表示。**ログ／結果では従来どおり正式な役名**を表示できます。
- **テーブル中央**: **ポットのみ**を **小型フェルトプレート**に表示（**サード／フォースストリート等のフェーズ名はプレイ画面に出さない**）。ポットの数字は **やや大きめ**。内部フェーズ管理・デバッグのフェーズ表示は従来どおり。
- **ヘッダ・ステータスバー**: **左**にアンティ・ブラインド（Blind OFF 時は「ブラインド：OFF」）。**右寄せ**に CPU 人数。**所持チップは各席パネル**（あなた・CPU 共通で「〇〇 チップ」）。
- **ログ表示（ドック上段左）**: 上記「ログの役割」参照。チップ増減のみ。**`overflow-y: auto`**、**`clamp`・`vh`** で高さ可変。新しい結果が追加されたら **最新へスクロール**。
- **サポートAI（ドック上段右）**: **ハイ候補／ロー候補**を分け、**候補役名ごとに %** を表示（**まだ見えていないカード**を山札から引いて完成させる想定の **簡易サンプリング**。厳密な全探索ではない。**外部 API・AI 通信なし**）。長いときは **欄内スクロール**。フォールド／オールイン／ショーダウン後は短い固定文。**ゲーム画面上に「※簡易目安・外部APIなし」等の注記は出さない**（README にのみ記載可）。
- **詳細ログ（ログタブ）**: 全文ログ。アンティ・ポット作成・勝者などは日本語中心に整理済みの行が多いです。**ガイド**（`[ガイド]`）もこちら。CPU の思考 **理由**は詳細ログに出さず省略（`detailOnly`）。
- **操作ボタン（ドック下段右・日本語）**: **2 段・計 6 個**。**上段**：フォールド／チェック／コール。**下段**：シングル／ダブル／トリプル。**オールインはここにはない**。**ダブル／トリプル**は **追加支払い額**（`+N`）表示。
- **役表示（ドック下段左）**: **縦スクロールなし**（余白コンパクト）。左列＝ハイ、右列＝ロー（**「ハイ役」「ロー役」見出しなし**）。**オールインボタンはロー役ブロック右下**に配置（空きに重ねてよいが **役名の行と被らない**）。**現在該当のみ強調**。5 枚未満は強調なし（**「未確定」文言なし**）。ローは **5 TOP〜8 TOP / ローなし**。**ホイール表記は出さない**（実装は 5 TOP に寄せる）。
- **各席カード下 2 行枠**: **直近プレイ**（例：コール 20、フォールド、アンティ、ブラインド等）を **1 行目**、**2 行目**に **累計／直近（投入）** と **暫定ハイ／ロー**（条件を満たす場合のみ。Player は 5 枚以上で可。CPU はショーダウン後かつ全枚公開時のみ）。フォールド時は役の代わりに **フォールド**が分かる表示。**枠の高さは固定**。
- **下段レイアウト**: **CSS Grid** で **役名列**は必要幅を確保し、**操作ボタン列**は残り幅で拡張。人間席の表示は **「あなた」** のみ（**「あなた（あなた）」のような二重ラベルは付けない**）。
- **シングル**: UI 上の旧「ベット」。このラウンドで **まだ誰も張っていない**（`target===0` かつ自分の投入 0）ときに、設定の **ベット単位**をオープンします。顔合わせ時は **コール**などを使用します。
- **ダブル／トリプル**: このラウンドの **現在の最大投入 `target`** を基準に、自分の投入を **2 倍／3 倍**の目標額まで引き上げます（**追加は目標との差分のみ**）。`target===0` のときは目標が **2×／3× ベット単位**です。CPU の **レイズ**も同じ **2 倍目標**ルールに合わせています。
- **各席のチップ表示**: パネル上部に **所持チップ（〇〇 チップ）**（**あなた・CPU 同形式**）。**累計**（`totalCommitted`）と **直近の支払額**は **カード下 2 行枠の 2 行目**に **`累計 N / 直近 M`** として表示。
- **席配置**: CPU 1〜5 をテーブル外周に寄せ、中央を小さくして **円形に近い配置**（360px 幅でも崩れにくいよう CSS 調整）。
- **チップ補充（仮想・換金不可）**: トップ／対戦設定／戦績の **「チップ補充（〜1000）」** で、**あなたのチップが 1000 未満のとき 1000 まで回復**（確認ダイアログあり）。**ハンド進行中は不可**。`localStorage` の戦績に保存。**課金・換金ではありません**。
- **チップ不足**: 新ハンド開始時に足りない場合はメッセージで **チップ補充** を案内。CPU 席がアンティ等に不足している場合は **ハンド開始時に初期チップへ戻す**処理があります。
- **デバッグ**: **ゲーム内の「デバッグ」タブ**（またはヘッダのデバッグボタン）でのみパネルを表示。トップに常時のデバッグパネルはありません。

### 以前からの要点

- **「次へ」の廃止**: ストリート間・結果は手動の「次へ」なしで進みます。
- **自分の番以外** は操作ボタンが無効化され、CPU 等が自動で進みます。

## 初心者向け：ゲーム開始までの流れ

1. **`index.html`** をブラウザで開く（再読み込みで必ず **トップ** が表示されます）。
2. **「ゲーム開始」** で **対戦設定** 画面へ。
3. CPU 人数・チップ・アンティ・ベットなどを選ぶ（変更はその都度 `localStorage` に保存されます）。
4. **「この設定で開始」** で **ゲーム画面** に移り、**アンティ・ブラインド（ON 時）・Third Street 配布** が自動で行われ、第 1 ベットラウンドへ進みます。
5. 自分の番になったら **フォールド／チェック／コール** と **シングル／ダブル／トリプル**で操作し、**オールイン**は **下部のロー役付近のボタン**から行います。進行の詳細は **ログ** タブ、ポット・サイドポットの内訳は **結果** タブを参照します。

### PWA として使う（任意）

1. プロジェクトを **ローカルサーバー** で開くと Service Worker が登録されやすいです。例（Python 3）:

   ```bash
   cd seven-stud-hilo-app
   python -m http.server 8080
   ```

2. ブラウザで `http://localhost:8080/` を開きます。
3. 「ホーム画面に追加」や「アプリをインストール」から **standalone** 表示で利用できる場合があります（ブラウザ依存）。
4. 初回読み込み後、オフラインでもキャッシュされた `index.html` / `style.css` / `app.js` などが読み込める構成です（`file://` 直開きでは Service Worker が使えないことがあります）。

アプリ名（manifest）: **スタッド・ハイロー トレーナー** / 短名: **スタッドHiLo**（表示名は manifest 参照）

## 主な機能

- **CPU 1〜5 人**（名前: CPU 1 … CPU 5）。ショーダウンは **フォールドしていない席のみ** 合算サマリーの勝者判定に使います（**各サイドポットでは eligible のみ**）。
- **設定**（`localStorage`）: CPU 人数、初期チップ、アンティ、ベット単位、**Blind ON/OFF**、**Small / Big Blind**、難易度、プレイ速度、**カード演出**、**効果音**、**フォールド確認**。トップの **設定リセット** または **ゲーム内デバッグタブ**から初期化可能。
- **ポット分配**: ポット層ごとにハイ／ロー／スクープ、ロー不成立時はハイが該当分を取得。タイは山分け（端数はハイ側プールを `ceil` する方式で調整。詳細は実装参照）。
- **戦績**（v3）・**直近 10 ハンド履歴**（ポット層の記録を含む）。
- **PWA**: `manifest.json` / `service-worker.js` / `icon.svg`

## 手動確認項目（リリース前チェック）

- 起動時に **トップ画面** が表示される。
- **この設定で開始** 後に **ゲーム画面** でプレイできる。
- **直近プレイ内容**が **各プレイヤーのカード下**に出る（日本語。Fold / Call / Check / All-in / Ante / Blind 等が分かる）。
- **カード下に 2 行分の固定枠**があり、内容が少なくても **高さが潰れず**、席の位置が大きく動かない。
- **役**が **カード下**に表示される（Player は 5 枚以上で可。**ショーダウン前の CPU 役は出ない**。ショーダウン後・フォールドしていない CPU のみ）。
- **下部上段左**の **ログ表示**は **`overflow-y: auto`** で縦スクロールでき、**チップ増減だけ**（`第Nハンド結果`・`席：±N`）。**プレイ内容・フェーズ名が混ざらない**。**Main Pot / Side / eligible 等の行は出ない**。**ハンド間は点線などで区切られる**。新規追加で **最新へスクロール**。
- **下部ログと役名エリア**の **padding / margin / gap** が抑えられ、**役名一覧はスクロールなし**で収まる（CPU 5 でも大きく崩れない）。
- **下部上段右**の **サポートAI**に **候補役名ごとの %**（**ハイ候補／ロー候補**）。**% は見えていないカードから引く想定**の目安。長いとき **欄内スクロール**。**「※簡易目安・外部APIなし」等が画面に出ない**。
- **下部下段左**：**オールインボタン**が **ロー役下の空き**（右下イメージ）にあり、**ロー役の文字に被らない**。**操作列にオールインがない**（**上段 3・下段 3 の計 6 ボタン**）。
- **ベット**表記が **シングル**。**シングル**で **ベット単位**がオープンされる（未オープン時）。
- **他プレイヤーが 20 投入済み**（`target=20`）のとき、**ダブル**の目標が **40**、**トリプル**が **60** になる。
- **他プレイヤーが 40 投入済み**のとき、**ダブル**目標 **80**、**トリプル** **120** になる。
- **既にラウンドで支払った分**は差し引かれ、**追加分だけ**支払われる。
- **「未確定」**が下部役に出ない。ロー不成立時 **ローなし**が強調される。
- **各プレイヤーパネル**に **所持チップ（〇〇 チップ）**があり、**あなたとCPUで形式が揃う**。**累計／直近**は **カード下**。**「あなた（あなた）」**のような重複がない。
- **アンティ・ブラインド**が **ヘッダ左**。**通常プレイ画面の中央にフェーズ名が出ない**。**ポット表示がやや大きい**。**CPU 1〜5** の円形に近い配置が維持される。
- **ショーダウン／フォールド決着後**、**結果タブへ自動切り替えしない**。**詳細・サイドポット内訳**は **結果タブ／ログタブ**。
- **チップ補充**・**デバッグタブ**・**Fold / All-in 確認**の既存挙動。
- **サイドポット**が **結果・詳細ログ**で追える。**All-in / 分配**が破綻していない。
- **履歴**・**localStorage**・**PWA** が壊れていない。
- JavaScript: `node --check app.js` が通る。

## 手動テスト手順（目安）

1. 設定で **CPU 1〜5** を切り替え、席配置が人数どおりになることを確認する。
2. **短いチップ**で **All-in** し、**サイドポット**が結果タブ・ログ（メインポット／サイドポット作成）に現れることを確認する。
3. **CPU 5・Blind ON** で **All-in / Fold** を含むハンドが最後まで進むことを確認する。
4. **効果音 / フォールド確認 / 新しいハンド** の既存挙動を確認する。
5. **ゲーム内デバッグ**タブを開き、**Side pot 構造テスト** がすべて **OK** になることを確認する。
